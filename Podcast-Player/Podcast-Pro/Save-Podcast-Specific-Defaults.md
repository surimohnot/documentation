Defaults let you save reusable settings for a specific podcast feed. Players for that feed inherit the defaults only when the individual player has not supplied its own value.

## Steps

1. Go to **Podcast Player > Toolkit > Defaults**.
2. Select a saved podcast.
3. Fill the default values you want to reuse.
4. Leave fields empty if they should not become defaults.
5. Click **Save Defaults**.

## Available Default Sections

- **General Defaults**: Episode count, offset, sorting, title filter, display style, subscription menu.
- **Podcast Content**: Cover image, external image URL, description, teaser text, excerpt length.
- **Show/Hide Player Items**: Header, list, search, title, cover, download, social, featured images, and other visibility settings.
- **Player Styling**: Accent color, background color, text scheme, Google font, crop ratio, crop position, grid columns.
- **Subscription Links**: Service-specific subscription URLs.
- **User Feedback**: Feedback prompt and response settings.

## Clear Defaults

1. Select the podcast.
2. Click **Clear Defaults**.
3. Confirm the defaults no longer apply.

## Priority Rule

Instance settings win. Defaults fill only missing, empty, or unchanged base values.
